# Princess v01 — Phases 1-4 Complete
**Date:** 2026-04-21
**Session length:** ~3 hours
**Blender version:** 5.0

## What got built tonight

Full Princess v01 mesh — head, body, legs, tail. First time she reads as a dog from any angle.

### Phase 1: Head
- UV sphere base (32×16) with Mirror Modifier workflow
- Snout extruded, positioned mid-face
- Standing ears as separate mirrored objects
- Decision: standing ears acceptable for v01 — Princess's real ears stand when alert

### Phase 2: Body
- Cylinder base, 12 vertices, oriented along Y axis (spine direction)
- Neck extruded forward toward head
- Back hip area extruded with droop for shape
- Half mesh deleted, Mirror Modifier applied with Clipping enabled
- Body thickened and shaped after initial extrusions

### Phase 3: Legs
- Local subdivisions on underside for leg attachment (smarter than global subdivide)
- Front legs: inset → extrude upper → loop cut for knee → extrude lower → scale bottom for paw
- Back legs: same process, attached at hip area
- Mirror Modifier built the other side automatically
- One knee loop ended up angled — ignorable for v01

### Phase 4: Tail
- Built as separate object (cylinder, 8 verts, rotated to Y axis)
- 4 tapered segments, slight upward angle
- Positioned with overlap into body for visual connection
- Not joined to body mesh — will attach via armature parenting in Phase 5

## Decisions made

- **Ears stay standing** — Princess's real ears often stand. Not breed-accurate but close enough for v01.
- **Separate objects over joined mesh** — head, body, tail, ears all stay independent. Rigging will tie them together.
- **Local subdivide instead of global** — keeps low-poly aesthetic, adds detail only where legs attach.
- **Paws = scaled bottom face, no toes** — low-poly aesthetic, no detail needed.
- **Swap-ready topology** — clean quads, standard positioning, ready for armature with standard bone names.

## What broke / lessons

- Mistook intentional leg-attachment subdivisions for topology errors — verified clean with Merge by Distance (0 verts removed).
- Confused myself on body orientation by misreading viewport gizmos — body oriented along Y is correct per original instructions.
- One loop cut ended up diagonal instead of horizontal — kept it, acceptable for v01.
- Head and neck gap closed by moving head object, not re-extruding neck.

## What's next

**Phase 5: Armature and rigging**
- Standard quadruped bone naming (root, spine, neck, head, tail chain, shoulder.L/R, thigh.L/R, etc.)
- Parent each mesh object to armature with Auto Weights
- Test pose mode on each bone chain
- Fix weight paint where deformation is ugly

**Swap-ready contract reminder:** when v02 mesh comes later, new mesh parents to same armature with same bone names, auto-weight, scenes that link Princess update automatically.
